# EasyLog セットアップマニュアル

> 代行セットアップ用手順書。1顧客あたり約45分で完了。

---

## 事前に顧客から聞くこと

| 項目 | 例 |
|------|-----|
| 旅の開始日 | 2026-01-28 |
| 希望するBot名 | @YukiTravelBot |

---

## STEP 1: Telegram Botの作成

1. Telegramで `@BotFather` を開く
2. `/newbot` を送信
3. Bot名を入力（例：`Yuki Travel Log`）
4. ユーザー名を入力（例：`YukiTravelBot`）※末尾に `bot` が必要
5. 発行された **Bot Token** をメモする
   ```
   例: 1234567890:AAFkqqcPEfBO1bupvI5yeWcfZ1RxJS8hSsU
   ```

### 顧客のTelegram User IDを取得する

1. 作成したBotに顧客からメッセージを送ってもらう
2. ブラウザで以下のURLにアクセス（Bot Tokenを置き換える）
   ```
   https://api.telegram.org/bot{BOT_TOKEN}/getUpdates
   ```
3. レスポンスの `message.from.id` の数値が **Telegram User ID**

---

## STEP 2: Supabaseのセットアップ

1. [supabase.com](https://supabase.com) にログインして **New Project** を作成
2. プロジェクト名・パスワードを設定（パスワードは保管しておく）
3. 作成完了後、左メニュー **Settings → API** から以下をコピー
   - **Project URL** → `SUPABASE_URL`
   - **anon public** キー → `SUPABASE_ANON_KEY`

### テーブル作成

左メニュー **SQL Editor** を開き、以下を実行：

```sql
create table posts (
  id uuid primary key default gen_random_uuid(),
  user_id text,
  text text,
  file_id text,
  image_url text,
  location_lat float,
  location_lng float,
  emotion text,
  created_at timestamptz default now()
);

create table articles (
  id uuid primary key default gen_random_uuid(),
  content text,
  date date,
  created_at timestamptz default now()
);

alter table posts disable row level security;
alter table articles disable row level security;
```

## STEP 3: Gemini APIキーの取得

> ※ 顧客自身のGoogleアカウントで取得してもらう（APIコストは顧客負担）

1. [Google AI Studio](https://aistudio.google.com) にアクセス
2. **Get API key → Create API key** をクリック
3. 発行された **APIキー** をメモする
4. [Google Cloud Console](https://console.cloud.google.com) で支払い情報を登録する
   - 無料枠内での使用だが、登録しないとAPI呼び出しが制限される

---

## STEP 4: Renderへのデプロイ

1. [render.com](https://render.com) にログイン（GitHubアカウントで可）
2. **New → Web Service** をクリック
3. GitHubリポジトリ `EasyLog` を選択して接続
4. 以下の設定を入力：

   | 項目 | 値 |
   |------|----|
   | Name | `easylog-{顧客名}` |
   | Region | Singapore（日本に近い） |
   | Branch | `main` |
   | Build Command | `npm install` |
   | Start Command | `node src/index.js` |
   | Instance Type | Free |

5. **Environment Variables** に以下を設定：

   | キー | 値 |
   |------|----|
   | `TELEGRAM_BOT_TOKEN` | STEP1で取得したToken |
   | `TELEGRAM_WEBHOOK_URL` | （後で設定。今は空白） |
   | `SUPABASE_URL` | STEP2で取得したURL |
   | `SUPABASE_ANON_KEY` | STEP2で取得したキー |
   | `GEMINI_API_KEY` | STEP3で取得したキー |
   | `TELEGRAM_USER_ID` | STEP1で取得したID |
   | `TRIP_START_DATE` | 旅の開始日（例: `2026-01-28`） |
   | `PORT` | `3000` |

6. **Create Web Service** をクリック
7. ログに `Server running on port 3000` が出るまで待つ（約3分）

---

## STEP 5: Webhookの設定

デプロイ完了後、RenderのサービスURLを確認する（例：`https://easylog-yuki.onrender.com`）

1. Renderの **Environment Variables** を開く
2. `TELEGRAM_WEBHOOK_URL` に Render の URL を設定
   ```
   例: https://easylog-yuki.onrender.com
   ```
3. **Save Changes** → 再デプロイが始まる
4. 再デプロイ完了後、以下のURLでWebhook確認：
   ```
   https://api.telegram.org/bot{BOT_TOKEN}/getWebhookInfo
   ```
   `"url": "https://...onrender.com/webhook"` が表示されればOK

---

## STEP 6: 動作確認

### Botへのメッセージ確認
1. 顧客のTelegramでBotにメッセージを送る
2. 「記録しました！」と返信が来ればOK
3. 写真+キャプションを送り「画像を記録しました！」が来ればOK

### 記事生成の確認
1. Telegramで `/generate` を送信
2. 「記事を生成中です...」が返ってきた後、記事が届けばOK

## STEP 7: 顧客への引き渡し

引き渡し物：

- [ ] Telegram BotのURL（例：`https://t.me/YukiTravelBot`）
- [ ] 使い方説明（下記）

### 顧客向け使い方説明

```
【EasyLog 使い方】

■ 記録方法
・テキスト → そのまま送る
・写真 → キャプションをつけて送る（キャプションが記事に反映されます）
・位置情報 → 添付ボタンから「位置情報」を選択

■ 記事生成
・毎日22:00 JST に自動生成
・手動で生成したい場合は /generate を送る

■ 出力
1. 写真（あれば順番に送信）
2. note用の旅行記事
3. X/SNS用の投稿テキスト（500文字以内）
```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| Botが反応しない | サーバースリープ or Webhook未設定 | STEP5を再確認 |
| 「保存に失敗しました」 | Supabase接続エラー | 環境変数を確認 |
| 記事生成でエラー | Gemini APIキーの問題 | 支払い情報登録を確認 |
| 「本日の投稿がありません」 | 時差によるUTCの日付ズレ | 9時以降の投稿で再テスト |
| 記事内容が旅行と無関係 | 写真のみでキャプションなし | 写真にキャプションをつけるよう案内 |
