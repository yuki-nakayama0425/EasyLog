# EasyLog LP デザイン企画書

---

## 1. コンセプト

**「旅人が旅しながら読む、旅人のためのLP」**

派手に売り込まない。ユーザーの「あ、これ欲しかったやつ」を引き出す。
旅の空気感をまとった、シンプルで没入感のあるデザイン。

---

## 2. ターゲット

| 項目 | 内容 |
|------|------|
| 年齢 | 20〜30代 |
| 属性 | 世界一周・長期旅行中または検討中のバックパッカー |
| デバイス | **スマホメイン**（旅中にSNSで見つけるケースが多い） |
| 悩み | 発信したいけど文章が書けない・時間がない |
| 欲求 | 旅の記録を資産にしたい・フォロワーを増やしたい |

---

夜の空港 / 旅の余韻 / シンプル / 没入感

## 4. ページ構成（セクション）

---

### SECTION 1 : Hero（ファーストビュー）

**目的：** 3秒で「これは自分に必要だ」と思わせる

**構成：**
```
[背景：夜の旅先の街並み or 星空の写真（暗めのオーバーレイ）]

　　　EasyLog

　つぶやくだけで、旅が記事になる。

　旅の記録をTelegramに送るだけで
　毎日自動でnote記事とX投稿文を生成。
　PCも文章力も不要。

　[今すぐ始める →]　[サービスの詳細を見る]
```

---

### SECTION 2 : 課題提起（共感）

**目的：** 「わかる」と思わせる

**構成：**
```
こんなことで困っていませんか？

😔 旅の記録を残したいけど、文章を書く時間がない
😔 写真はいっぱいあるのに、記事にまとめられない
😔 発信したいのに、毎日続けられない
😔 旅が終わってから書こうとすると、記憶が薄れる
```

---

### SECTION 3 : 解決策（What is EasyLog）

**目的：** サービスの本質を一言で伝える

**構成：**
```
EasyLogは、旅の"つぶやき"を記事に変えるボットです。

Telegramに送るだけ。あとは全部自動。

[図解：Telegram送信 → 自動生成 → note/X に届く]
```

---

### SECTION 4 : 使い方（How it works）

**目的：** シンプルさを視覚的に伝える

**構成：**
```
使い方は3ステップだけ

STEP 1  📱
旅先でTelegramに送る
「象がいた」「カレーうまかった」
写真＋一言キャプションでOK

STEP 2  🤖
毎晩22時に自動で生成
AIが1日のログをもとに
旅行記事を自動作成

STEP 3  📝
コピペしてnote/Xに投稿
note用記事とX用テキストが
そのまま使える形で届く
```

---

### SECTION 5 : サンプル記事（実績）

**目的：** 「こんな記事が作れるの？」という驚きを与える

**構成：**
```

・象いた
・カレーうまかった
・宿が廃墟だった

[生成された記事のスクリーンショット or テキスト表示]

【世界一周85日目・スリランカ】
予約した宿が廃墟だった日、
それでも紅茶はうまい。
...
```

---

### SECTION 6 : 機能一覧（Features）

**目的：** 細かい機能で「これは使える」と確信させる

**構成：**
```
✅ テキスト・写真・位置情報を記録
✅ 毎日22:00に自動生成（手動生成も可）
✅ note用記事（800〜1300文字）を出力
✅ X用投稿文（500文字以内）を出力
✅ 「世界一周〇〇日目」タイトル自動付与
✅ Googleマップリンクを記事に自動挿入
✅ スマホだけで完結。PC不要
✅ 24時間稼働。寝ている間も動いてる
```

## 5. スマホ対応方針

- フォントサイズ：本文16px以上
