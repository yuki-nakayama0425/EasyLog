## 1. 概要

TravelLog AIは、日中に投稿されたテキスト・画像・位置情報をもとに、1日の終わりに自動で旅行記（note形式）を生成するAI日記サービスである。

ユーザーはTelegram Botを通じて投稿するだけで、構造化された記事が生成される。

---

## 2. 目的

優先順位：

1. 日記作成の省力化
2. 発信コンテンツ生成（note用）
3. 記録の資産化（振り返り・分析）

---

## 3. ターゲット

- 海外バックパッカー
- 個人旅行者
- SNS発信に興味があるユーザー

---

## 4. システム構成

### フロント
- Telegram Bot

### バックエンド
- Node.js（推奨） or Python

### DB
- Supabase（PostgreSQL）

### AI
- OpenAI API

### インフラ
- Vercel / Cloud Functions

---

## 5. 機能一覧

### 5.1 投稿機能（Telegram Bot）

#### 入力
- テキスト（数行）
- 画像
- 位置情報（任意）
- 感情タグ（任意）

#### 投稿頻度
- 1日最大10〜20件想定

---

### 5.2 データ保存

#### テーブル設計（posts）

| カラム名 | 型 | 説明 |
|--------|----|------|
| id | uuid | PK |
| user_id | text | Telegram ID |
| text | text | 投稿内容 |
| image_url | text | 画像URL |
| location_lat | float | 緯度 |
| location_lng | float | 経度 |
| emotion | text | 感情タグ |
| created_at | timestamp | 投稿時間 |

---

### 5.3 画像処理

- Telegramから画像取得
- ストレージ（Supabase Storage等）に保存
- 圧縮処理（任意）
- AIでキャプション生成

---

### 5.4 記事生成機能（コア）

#### 実行タイミング
- 毎日夜（cron）

#### 処理フロー

1. 当日投稿を取得
2. 時系列ソート
3. 画像キャプション生成
4. AIで記事生成
5. テキスト整形

---

### 5.5 出力仕様

#### フォーマット（note用）

- タイトル
- 導入文
- 本文（時系列）
- まとめ
- ハッシュタグ

#### 出力形式
- プレーンテキスト
- Telegramに返信 or Web画面表示

---

## 6. AIプロンプト設計

### 入力データ例
日時: 2026-04-20 10:00
内容: 朝ごはんにカレーを食べた。めちゃくちゃ美味しい。
画像: curry.jpg
場所: Colombo
感情: happy

---

### プロンプト


以下は1日の旅行ログです。
これをもとに、note風の旅行記を作成してください。

要件：
・時系列で整理
・自然なストーリーにする
・感情を補完する
・読者が旅行したくなる内容
・エモい文章

構成：
・タイトル
・導入文
・本文
・まとめ
・ハッシュタグ

## 7. API設計

### 7.1 Telegram Webhook

POST /webhook

#### 処理
- メッセージ解析
- テキスト抽出
- 画像取得
- DB保存

---

### 7.2 記事生成

POST /generate

#### 処理
- 当日データ取得
- OpenAI API呼び出し
- テキスト生成
- 保存 or 返却

---

## 8. cron設計

- 1日1回（例：22:00）
- /generateを実行
